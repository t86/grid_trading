# 币安合约交易赛使用说明

本文档面向“按指定币种、指定时间窗口，尽量提升合约交易量并控制损失”的交易赛场景。

## 1. 目标与思路

交易赛常见目标：
- 在比赛窗口内提高成交量（通常按合约成交额排序）。
- 在提高成交量同时控制回撤与净亏损。

本系统在 `competition_volume` 目标下的排序逻辑是：
1. 先看目标成交量达成情况（`目标达成率`）。
2. 达成率相近时，优先损失更小/收益更高。
3. 同等风险收益下，优先更高成交额。

## 2. 快速开始（Web）

启动：

```bash
cd <repo>
PYTHONPATH=src python3 -m grid_optimizer.web --host 127.0.0.1 --port 8787
```

打开：`http://127.0.0.1:8787`

## 3. 推荐操作流程

1. 选择 `市场类型=futures` 与比赛币种。
2. 设置 `开始时间` 和 `结束时间`，精确对齐比赛窗口。
3. 设置 `最大投入资金` 与 `目标成交额`。
4. 目标函数选择 `competition_volume`。
5. 先点击 `智能建议 min/max`，挑一组建议区间并应用。
6. 再点击 `开始测算`，查看 `Top 候选` 与 `每格买入计划`。

## 4. 如何读“智能区间建议”

建议表关键字段：
- `推荐 N`：建议网格数。
- `推荐模式`：建议资金分配模式。
- `单格价差`：每格平均价差比例。
- `每格买入额(均/最小/最大)`：资金如何分布到网格。
- `成交额`：回测窗口内累计成交额。
- `目标达成率`：`成交额 / 目标成交额`。

实践建议：
- 达成率低：提高 `N` 上限、缩窄价格区间、改用 `1s` 或 `1m` 数据。
- 成交额高但亏损偏大：适当降低 `N`、放宽区间、降低方向性更强的模式权重。

## 5. 分配模式建议

交易赛更看重高换手和可控亏损，方向不确定时建议优先：
- `equal_qty`
- `equal`
- `center_heavy`

谨慎使用更偏方向性的模式：
- `linear`
- `quadratic`
- `geometric`

这些模式更适合你对单边方向有明确判断时使用。

## 6. 数据频率与缓存

- 支持 `1s` 数据：通过 Binance Futures `aggTrades` 聚合为 1 秒 K 线。
- K 线和资金费率都会按交易对与周期缓存到本地 `data/`。
- 首次拉取较慢，后续同区间或相邻区间会优先复用缓存。

## 7. 风险提示

- 回测不能覆盖全部实盘因素，例如盘口深度、滑点变化、资金费跳变。
- 比赛阶段建议滚动回测最近 `1d / 3d / 7d` 多个窗口交叉验证。
- 本工具用于研究与参数筛选，不构成投资建议。
