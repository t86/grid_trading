# 次日榜单预测设计

## 目标

在现有交易赛榜单页面上增加“次日榜单预测”能力。用户输入自己对下一天新增交易量的预估后，系统基于当前榜单、历史日榜趋势、比赛剩余时间，给出：

- 次日总榜量预测
- 次日关键名次门槛预测（1/5/20/50/200）
- 已录入参赛用户的次日累计量、预测排名、奖励区间和追赶差距
- 保守 / 基准 / 激进三种情景

## 范围

只在现有 `competition_board` 页面和相关 API 上扩展，不新建独立服务。

本次实现不做：

- 机器学习模型
- 盘口级或分钟级细粒度预测
- 自动下单或自动调策略

## 数据来源

- 当前榜单快照：`build_competition_board_snapshot()`
- 历史日榜趋势：`load_competition_board_trend()`
- 用户录入：`competition_board_entries.json`

## 预测方法

### 总量与名次门槛

对最近两天日榜增量做加权：

- `base_delta = 0.65 * 昨日增量 + 0.35 * 前日增量`

再按比赛阶段施加尾盘因子：

- 倒数最后一天：乘以更高的尾盘因子
- 倒数第二天：乘以较小的尾盘因子
- 其余时间：不额外放大

尾盘因子还会参考最近两天的增量加速度，但会做上限裁剪，避免最后一天预测失控。

### 用户次日预测

用户输入“下一天预估交易量”后：

- `predicted_user_total = 当前录入值 + next_day_volume`

为得到预测排名，需要构造一份“次日预测榜单”：

- 保留当前榜单的可见行
- 按关键名次门槛的预测增量，对当前各行做分段线性平移
- 再复用现有 `_entry_projection()` 逻辑给出预测排名 / 奖励段

### 情景输出

统一输出三档：

- `conservative`
- `base`
- `aggressive`

其中：

- 保守：较低尾盘因子和较低日增量
- 基准：标准加权增量
- 激进：更高尾盘因子和更高日增量

## 页面入口

在现有交易赛榜单页的“参赛人员录入”下方增加一个“次日榜单预测”卡片：

- 选择榜单
- 选择已录入用户（可空）
- 输入“下一天预估交易量”
- 点击预测
- 显示三种情景结果

## 验证

除了单元测试外，用真实的：

- `OPN`
- `ROBO`

历史趋势数据做一次真实校验，确认接口能在已有历史数据上返回合理结果。
